# Nearsited Project Summary (Updated)

## Product Positioning

Nearsited helps web designers and agencies find businesses that:

- Need a website
- Need a better website
- Depend too heavily on social media

The platform discovers opportunities, helps evaluate them, generates outreach, and helps track active prospects.

Avoid internal positioning such as:
- Opportunity Intelligence Platform
- Revenue Intelligence
- Category-creation terminology

Customer-facing language should remain simple and direct.

---

# Core Opportunity Types

The product supports three distinct opportunity types:

1. Weak Website
2. Social Only
3. No Website

These are foundational product concepts and should remain visible throughout the application.

---

# Landing Page Direction

## Goal

Visitors should understand within 5–10 seconds:

- What the product does
- Who it helps
- Why it is useful

## Key Messaging

Preferred:

- Find businesses that need websites
- Discover website opportunities
- Generate outreach
- Win website projects
- Find businesses with a stronger online presence

Avoid abstract terminology.

## Structural Changes

- Added Sample Opportunity List
- Replaced generic agency-size positioning
- Added Founder Story section (between Agency Use Cases and Objections)
- Added support for all opportunity types
- Simplified messaging

# Opportunity Detail Experience

Each opportunity type has its own workflow.

## Weak Website

- Audit-driven workflow
- Opportunity score
- Current vs potential score
- Business-focused summaries
- Outreach generation

## No Website

- No audit metrics
- No technical diagnostics
- Website opportunity framing
- Website-focused outreach

## Social Only

- Social presence analysis
- Website opportunity framing
- Social-aware outreach

# Pipeline Philosophy

Pipeline represents:

Businesses actively being pursued.

Not:

- Analysed businesses
- Generated pitches
- Discovered opportunities

## Rules

- Manual Add To Pipeline
- Prospect-first workflow
- Sales-focused stages

Stages:

- Prospect
- Contacted
- In Conversation
- Won
- Lost

---

# Dashboard Philosophy

The dashboard should answer:

"What should I do next?"

## Focus

- Ready To Pitch
- Active Prospects
- Conversations
- Pipeline Progress

Not:

- Large vanity counts
- Opportunity hoarding

Dashboard should encourage progress, not collection.

---

# Pitches Workspace

Pitches are an outreach workspace.

Not an archive.

Users should be able to:

- Understand why a pitch exists
- Copy outreach quickly
- Move opportunities into pipeline
- Work across all opportunity types

---

# Quick Site Audit (`/dashboard/audit`)

Purpose:

Show value quickly — audit any website without saving it as a lead.

Flow:

Paste Website URL
→ Run PageSpeed Audit (mobile + desktop)
→ Review scores and issues
→ Generate Outreach or Save as Lead

Remain intentionally minimal.

---

# Product Principles

1. Simplicity over sophistication
2. Customer language over startup jargon
3. Sales workflows over analytics
4. Actionable next steps
5. Opportunity-specific experiences
6. Minimal complexity
7. Fast path to value
8. Easy to understand

---

The goal is not feature completeness.

- Find businesses that need websites
- Generate outreach
