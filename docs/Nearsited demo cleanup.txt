## NOTES ON CAPTURING

**You don't need 70 separate screenshots.** Many of these can be captured in groups:
- Filter tabs (item 22) can be 7 quick screenshots in a row
- Dropdown open states can be captured as a group

**Screen recording is faster than screenshots** for flows with multiple steps (items 50, 55, 32). A short Loom or screen recording for those saves time.

**Priority within batches:** Items marked in **bold** are the highest priority within each batch — if you can only do a few, do those.

**What to look for while capturing:**
- Does the button change state after clicking (loading indicator, disabled, label change)?
- Does the action produce a toast, navigation, or inline update?
- Is there a visual difference between success and failure states?
- Does reloading the page preserve the action's result?
